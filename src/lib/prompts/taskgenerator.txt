Your goal is to generate "warm-up" tasks for a trainee based on a specific Emotion and a list of Descriptors.

Input:
- Emotion: [The target emotion, e.g., "Joyful" or "Homicidal"]
- Descriptors: [List of adjectives describing the state]

Output:
You must output a SINGLE JSON object containing a list of training tasks.
Do NOT output any markdown formatting (like ```json ... ```). Output RAW JSON only.

JSON Structure:
{
  "tasks": [
    {
      "type": "multiple_choice",
      "question": "string (Scenario or question related to the emotion)",
      "options": ["string", "string", "string", "string"],
      "correct_answer": "string (Must be one of the options)"
    },
    {
      "type": "true_false",
      "question": "string (Statement about dealing with this emotion)",
      "answer": boolean (true or false)
    },
    {
      "type": "fill_in_blank",
      "question": "string (Sentence with a blank represented by _____)",
      "correct_answer": "string (The word/phrase that fits)",
      "options": ["string", "string", "string", "string"] (List of 4 words including the correct answer)
    },
    {
      "type": "match_definition",
      "pairs": [
        {"term": "string", "definition": "string"},
        {"term": "string", "definition": "string"},
        {"term": "string", "definition": "string"}
      ]
    }
  ]
}

Requirements:
1. Generate at least 10 tasks. Mix the types up thoroughly.
2. The content must be relevant to the provided Emotion and Descriptors.
3. For "match_definition", provide exactly 3 or 4 pairs of terms/definitions.
4. For "negative" or "dangerous" emotions (e.g., Suicidal, Homicidal), ensure the questions reinforce safety, empathy, and proper intervention techniques.
4. For "positive" emotions, focus on recognizing and reinforcing that state.
5. The language should be professional yet accessible.
